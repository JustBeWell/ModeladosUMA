% ESTA VERSION NO TIENE NÚMEROS EN LOS TÍTULOS     Section, Subsection... 
\newcommand{\numberedsection}[1]{%
  \stepcounter{section}%
  \section*{#1}%
  \addcontentsline{toc}{section}{\protect\numberline{\thesection}#1}%
}
\newcommand{\numberedsubsection}[1]{%
  \stepcounter{subsection}%
  \subsection*{#1}%
  \addcontentsline{toc}{subsection}{\protect\numberline{\thesubsection}#1}%
}
\newcommand{\numberedsubsubsection}[1]{%
  \stepcounter{subsubsection}%
  \subsubsection*{#1}%
  \addcontentsline{toc}{subsubsection}{\protect\numberline{\thesubsubsection}#1}%
}

\begin{abstract}
  Esta práctica aborda el diseño e implementación de un sistema orientado a objetos para 
  gestionar un refugio de animales utilizando Java, siguiendo los conceptos de diseño orientado 
  a objetos presentados en el Tema 5.\par
  \vspace{0.5cm}
  El objetivo principal es analizar y comparar diferentes estrategias de diseño para implementar 
  este modelo, asi como casos especiales donde un mismo socio pueda desempeñar múltiples 
  roles simultáneamente. Se justifica por qué las clases descritas inicialmente no pueden 
  ser implementadas directamente en Java debido a la limitación de herencia simple. 
  Asimismo, se propone una solución basada en composición e interfaces discutiendo la decision
  tomada.\par
  \vspace{0.5cm}
  Finalmente, se presenta una solución acompañada de un nuevo diagrama de diseño que muestra 
  la arquitectura propuesta, con las relaciones implementadas, validaciones con \emph{assert} 
  para comprobar restricciones en el nuevo código, asi como métodos actualizados a 
  las necesidades presentadas para el caso especial de un Socio con multiples roles\par
\end{abstract}
